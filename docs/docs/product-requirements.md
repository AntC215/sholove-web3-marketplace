# ShoLove Product Requirements

## Objective

Build a Web3 digital asset marketplace where users can create interactive floating island storefronts to showcase, sell, and purchase NFTs or digital assets.

---

## Core Requirements

- Users should be able to connect a wallet.
- Users should be able to view NFTs connected to their wallet.
- Users should be able to create a personalized marketplace island.
- Users should be able to display selected assets on their island.
- Users should be able to list digital assets for sale.
- Buyers should be able to view asset details.
- Buyers should be able to purchase listed assets.
- The platform should support NFT metadata.
- The platform should support ERC-721 and ERC-1155 concepts.
- The interface should be clean, visual, and easy to understand.

---

## Marketplace Requirements

- Wallet connection
- Asset discovery
- Asset detail pages
- Listing creation
- Listing removal
- Purchase flow
- Seller payout concept
- Marketplace fee concept
- Transaction confirmation
- User storefront page

---

## User Experience Requirements

- Simple wallet connection
- Clear asset cards
- Visual marketplace layout
- Personalized island storefront
- Easy listing workflow
- Clear purchase confirmation
- Mobile-friendly design
- Beginner-friendly Web3 language

---

## Data Requirements

- Wallet address
- NFT contract address
- Token ID
- Asset name
- Asset image
- Asset description
- Metadata URI
- Listing price
- Seller address
- Buyer address
- Transaction hash
- Listing status
