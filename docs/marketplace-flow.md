# ShoLove Marketplace Flow

## 1. Wallet Connection

The user connects their wallet using MetaMask or WalletConnect.

The platform should verify:

- Wallet address
- Network
- Connection status
- Supported chain

---

## 2. Asset Detection

After the wallet is connected, the platform checks for digital assets connected to the wallet.

The platform may review:

- ERC-721 NFTs
- ERC-1155 NFTs
- NFT metadata
- Asset images
- Asset names
- Token IDs
- Contract addresses

---

## 3. Island Storefront Creation

The user creates a personalized digital storefront known as a floating island.

The storefront may include:

- User display name
- Wallet address
- Featured assets
- Island theme
- Collection sections
- Asset showcase areas

---

## 4. Asset Listing

The user selects an asset and creates a listing.

Listing data may include:

- NFT contract address
- Token ID
- Listing price
- Seller wallet
- Listing status
- Marketplace fee
- Metadata

---

## 5. Marketplace Browsing

Buyers can browse marketplace islands and view digital assets.

Buyers should be able to review:

- Asset image
- Asset name
- Description
- Price
- Seller
- Contract address
- Token standard
- Ownership status

---

## 6. Purchase Flow

The buyer selects an asset and confirms the purchase through their connected wallet.

The purchase flow may include:

- Wallet confirmation
- Smart contract purchase function
- Transfer of NFT ownership
- Seller payout
- Marketplace fee
- Transaction hash
- Confirmation state

---

## 7. Post-Purchase Experience

After purchase, the platform should update:

- Asset ownership
- Listing status
- Buyer collection
- Seller activity
- Transaction history
